# FIRE Simulator — Medwealth Lab Tokyo

**Human Capital Safety Valve 論文準拠シミュレーター**

高流動性職種（医師・ITエンジニア等）の専門職が「暴落時に一時的に就労復帰する」という選択肢（人的資本のセーフティバルブ）を織り込むことで、従来の4%ルールより高い取り崩し率でFIREが成立するかを検証するツールです。

🔗 **アプリURL**: https://fire-simulator-9wskcnzgfc883iy42yketm.streamlit.app

## 機能一覧（タブ別）

### 🏠 Tab 1 — 基本FIREシミュレーション（無料・全公開）

個人の財務情報を入力してFIRE到達年齢と資産推移を試算します。

**入力項目**

| 項目 | 説明 |
|------|------|
| 現在の年齢・年収・資産 | 基本プロフィール |
| 月間生活費・遊興費 | 年間支出の算出に使用 |
| 子供の年齢 | 教育費の現在価値（PV）を自動計算 |
| リスク許容度（低/中/高） | S&P500 / NASDAQ100 の比率を自動設定 |
| 暴落時に働ける職種か | `はい` → 取り崩し率6%、`いいえ` → 4% |
| 利回り・インフレ率 | 自動（リスク連動）または手動設定 |
| 月間投資額 | 自動（収入－支出）または手動設定 |
| 教育費（小中高大） | 公立/私立/手動入力を選択 |

**出力項目**

- FIRE到達年齢・必要資産額・手取り年収・教育費PV
- 利回り・インフレ率・取り崩し率・月間取り崩し額
- 資産推移グラフ（〜100歳）
- 年齢別資産テーブル（5年刻み）

> **取り崩し率の根拠**: S&P500平均リターン約11.8% − インフレ約2% − 安全マージン約2% ≒ 7%、保守的に6%設定。税金・取引コストはNISA枠を前提として除外。

### 🛡 Tab 2 — 人的資本セーフティバルブ（パスワード必要）

論文準拠のシミュレーションで「人的資本あり」と「ベースライン（なし）」の成功率を比較します。

**入力項目**

| 項目 | 説明 |
|------|------|
| 初期資産 | シミュレーション開始時の資産額 |
| 取り崩し率 | 4〜10%をスライダーで選択 |
| ポートフォリオ | 株/債券比率（100/0、75/25、50/50） |
| 人的資本モデル | Complete（取り崩しゼロ）/ Half（50%）/ Baseline（なし） |
| 閾値 | 就労復帰トリガー（生活費の5/7/10年分） |
| シミュレーション手法 | ローリングウィンドウ / モンテカルロ（t分布 df=5） |
| シミュレーション期間 | 10〜50年（論文推奨：30年） |

**出力項目**

- 成功率（HC・Baseline）・中央値最終資産・平均就労復帰年数・平均最大DD
- 資産推移パーセンタイル帯グラフ（10/25/50/75パーセンタイル）
- 取り崩し率別成功率比較グラフ（4〜10%）

**論文推奨シナリオ（ワンクリック反映）**

| シナリオ | 取り崩し率 | ポート | モデル | 特徴 |
|----------|-----------|--------|--------|------|
| Scenario A | 6% | 100/0 | Complete | 効率重視・フルタイム復帰 |
| Scenario B | 5% | 100/0 | Half | 部分復帰（パートタイム） |
| Scenario C | 5% | 75/25 | Complete | 保守型 |
| 4%ルール | 4% | 75/25 | Complete | Trinity Study基準 |

> **シミュレーション仕様**: Shillerデータ1926–2022、ローリングウィンドウ＋モンテカルロ（Student t分布 df=5、n=1,000、seed=42）

**入力項目**

| 項目 | 説明 |
|------|------|
| 退職開始年 | 1926〜1992年から選択 |
| 取り崩し率 | 4〜10% |
| 初期資産 | 万円単位で入力 |
| ポートフォリオ | 株/債券比率 |
| 人的資本モデル | Complete / Half / Baseline |
| 閾値 | 就労復帰トリガー（5/7/10年分） |
| シミュレーション期間 | 10〜50年 |

**出力項目**

- 最終資産・初期資産比・就労復帰年数・最低資産
- 就労復帰タイミング（年次バー表示：🟧=就労復帰、⬜=退職継続）
- 資産推移グラフ（就労復帰期間をオレンジ帯で強調）＋動的閾値ライン
- 歴史的イベント警告（大恐慌・スタグフレーション・リーマン等）

### 🔬 Tab 5 — CDR閾値研究（パスワード必要）

**CDR（Contribution Dependency Ratio）= 年間追加投資 ÷ 総資産**

「積み立てをいつやめても1年以内の遅れで済むか」を定量化するリサーチツールです。CDRは資産が複利で成長するにつれ時間とともに単調に低下します。

**入力項目**

| 項目 | 記号 | 説明 |
|------|------|------|
| 初期資産 | W₀ | 積み立て開始時点の保有資産（万円） |
| 年間追加投資 | C | 毎年の積み立て額（万円） |
| 期待リターン | r | 年率リターン・名目（%） |
| FIRE目標額 | W_target | 4%ルール基準：年間支出 × 25（万円） |

**出力メトリクス（5枚）**

| メトリクス | 説明 |
|-----------|------|
| 継続FIRE達成年数 | 積み立てを続けた場合のFIRE達成までの期間 |
| CDR閾値（遅れ1年以内） | このCDR以下になったら停止してよい水準 |
| 停止時の資産目安 | CDR閾値に対応する停止時の資産額 |
| 積み立て停止までの期間 | 今から何年後に停止してよいか（今すぐ可能な場合は「0年」表示） |
| 今すぐ停止した場合の遅れ | 現時点で停止した場合のFIRE達成遅延年数 |

**グラフ（3種）**

1. **CDR (%) vs 遅延年数 ΔT**
   - 横軸：CDR（%）、縦軸：FIRE達成遅延年数（年）
   - CDRが低い（資産が育った）ほど遅延が減る右下がり曲線
   - 緑の安全領域・赤のCDR閾値縦線・オレンジの遅れ1年ライン

2. **積み立て停止までの期間 t* vs 遅延年数 ΔT**
   - 横軸：停止時期（年）、縦軸：遅延年数（年）
   - 「あと何年積み立てれば停止して良いか」を時間軸で確認

3. **停止時の資産 W_stop vs 遅延年数 ΔT**
   - 横軸：停止時の資産額（万円）、縦軸：遅延年数（年）
   - 「いくらまで積み立てれば停止して良いか」を資産額で直接確認
   - 現在のW₀縦線・FIRE目標額縦線・停止時の資産目安縦線を表示

**使用数式**

$$T_{cont} = \frac{\ln\left(\frac{W_{target} + C/r}{W_0 + C/r}\right)}{\ln(1+r)}, \quad
t^* = \frac{\ln\left(\frac{C/\rho + C/r}{W_0 + C/r}\right)}{\ln(1+r)}, \quad
\Delta T = \max(T_{stop} - T_{cont},\ 0)$$

## GitHubへのアップロード手順

### Step 1：リポジトリを作成
1. https://github.com にログイン
2. 右上の「+」→「New repository」
3. Repository name: `fire-simulator`
4. **Private**を選択（コードを非公開にする場合）
5. 「Create repository」をクリック

### Step 2：ファイルをアップロード
1. 作成したリポジトリのページで「uploading an existing file」をクリック
2. 以下のファイルをドラッグ&ドロップ：
   - `app.py`
   - `simulation.py`
   - `requirements.txt`
3. **⚠️ `.streamlit/secrets.toml` はアップロードしないこと**
4. 「Commit changes」をクリック

### Step 3：.streamlit/config.toml のアップロード
1. 「Add file」→「Upload files」
2. `.streamlit/config.toml` のみアップロード
3. Commit

### app.py の更新方法（日常運用）
1. GitHub上のリポジトリで `app.py` をクリック
2. 右上の鉛筆アイコン（Edit）をクリック
3. 全選択（Ctrl+A）→ 新しいコードを貼り付け
4. 「Commit changes」をクリック
5. Streamlit Cloud の管理画面で「Reboot app」を押して反映

## Streamlit Community Cloud デプロイ手順

### Step 1：アカウント作成
1. https://share.streamlit.io にアクセス
2. 「Sign up」→ GitHubアカウントでログイン

### Step 2：アプリをデプロイ
1. 「New app」をクリック
2. Repository: `あなたのGitHubユーザー名/fire-simulator`
3. Branch: `main`
4. Main file path: `app.py`
5. 「Advanced settings」をクリック

### Step 3：Secretsを設定（重要）
「Advanced settings」の「Secrets」欄に以下を貼り付け：

```toml
PAID_PASSWORD = "0216"
```

6. 「Deploy!」をクリック

### Step 4：公開URL確認
数分後に `https://あなたのアプリ名.streamlit.app` という形でURLが発行されます。
